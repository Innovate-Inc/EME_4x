EPA Metadata Editor (EME)

Interface Element Description: Tab 1

Element: Temporal Extent of Resource
Type: compound (single date, range of dates, or indeterminate time periods, and optionally time interval)
ISO Element Name: EX_TemporalExtent

EPA Requirement/Guidance

This element is required for EPA geospatial assets as per minimum FGDC standards. Users may choose to enter a single date or a range of dates depending on the time period of the data set/resource. Use the single date option for documenting the time period information when the resource reflects a single date. Use a range of dates for documenting the time period information when the information in your resource reflects a range of dates. Use indeterminate dates to document unknown dates and present dates.

To document the date information using the EME, a user would enter the information using one of the following options:

  1. Single Date: Select the Single Date tab and enter a single date using the calendar feature. Or choose an indeterminate date value from the drop down list. The read-only text box will show the to be committed to the metadata record on save.
  2. Range of Dates: Select the Time Period Table and use the calendar feature to enter begin and end dates. Or choose an indeterminate date value from the drop down list. Users can optionally specify a Time Interval to represent the frequency between time events based on floating point values. Time Intervals contain an integer entered in the first text box followed by the type of units in the second text box. (e.g., 6 Days)

Dates should be formatted as pairs of {start datetime/end datetime} in theISO 8601 format. ISO 8601 specifies that datetimes can be formatted in a number of ways, including a simple four-digit year (eg. 2013) to a much more specific YYYY-MM-DDTHH:MM:SSZ, where the T specifies a seperator between the date and time and time is expressed in 24 hour notation in the UTC (Zulu) time zone. (e.g., 2011-02-14T12:00:00Z/2013-07-04T19:34:00Z). Use a solidus ("/") to separate start and end times.

If there is a need to reflect that the dataset is continually updated, ISO 8601 formatting can account for thiswith repeating intervals. For instance, updated monthly starting in January 2010 and continuing through the present would be represented as:R/2010-01/P1M.

Updated every 5 minutes beginning on February 15, 2010 would be represented as: R/2010-02-15/PT5M.

ISO Definition

Time period covered by the content of the dataset

FGDC Definition

Information about the date and time of an event.

Domain

  • YYYY-MM-DD
  • Indeterminate Time Periods: “unknown”, “after”, “before”, and “now”
