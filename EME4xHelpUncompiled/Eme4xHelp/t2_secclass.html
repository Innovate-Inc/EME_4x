EPA Metadata Editor (EME)

Interface Element Description: Tab 2 - Security Constraints

Mandatory element

EPA Requirement/Guidance

This section pertains to the restrictions applied to the resource or metadata to protect security concerns. Use Limitation and Classification are required for minimum EPA compliancy so that assets may be properly categorized with respect to sharing with external agencies and the general public. The following elements are included:

Use Limitation/Access Level - Degree to which this resource could be made available to the public, regardless of whether it is currently available to the public. This element is mandatory. Best practices: If a member of the public can walk into your agency and obtain a data resource, that entry is public even if there are no files online. A restricted public data resource is one only available under certain conditions or to certain audiences (such as researchers who sign a waiver). A non-public data resource is one that could never be made available to the public for privacy, security, or other reasons as determined by your agency. This element is Mandatory.

Classification - Name of the handling restrictions on the resource or the metadata. This element is Mandatory.

User Note/Access Level Comment - An explanation of the classification level applied to the resource or metadata. This element is Mandatory only if Access Level is not "public". Free text is OK if Access Level is "public", but otherwise please choose from the standard list of categories for why access to a dataset might be restricted.

Classification System - Name of the security classification system.

Handling Description - Additional information regarding security restrictions on handling the resource or metadata.
