EPA Metadata Editor (EME)

Interface Element Description: Tab 2

Element: Longitude Resolution
Type: real
Short Name: longres

FGDC Definition

The minimum difference between two adjacent longitude values expressed in Geographic Coordinate Units of measure. The purpose of the latres and longres fields is to indicate how coarsely or finely information was recorded. The longres value represents the minimum possible difference between coordinate values.

EPA Requirement/Guidance

This element is not required for minimum FGDC compliancy, but becomes required when the "Geographic" element is used.

Note: The X field in EME corresponds to the Longitude Resolution FGDC element.

The purpose of the longres field is to indicate how coarsely or finely information was recorded. This information is usually captured when one uses the EPA Synchronizer to synchronize the coordinate system information from the data set. When one uses the EPA Synchronizer, the latres and longres fields will be automatically populated with the correct information from the dataset. The EPA Synchronizer is the best way to capture this information, and it is recommended where possible for documenting this element. One can attempt to do this calculation manually however, using ArcMap or another GIS program. Calculating this information in ArcMap may require a few steps. The longres value represents the minimum possible difference between coordinate values. One will need to perform a function that compares all of the points or vertices within the data set and then determine the minimum distance among all of them. This is best done using a function such as the "near" function (ArcPy) or another spatial processing function. One may also reference the ESRI help to review the "Resolution" property of a geodatabase feature class.
