EPA Metadata Editor (EME)

Interface Element Description: Tab 1

Element: Online Linkage
Type: text
Short Name: onlink

FGDC Definition

The name of an online computer resource that contains the data set.

EPA Requirement/Guidance

Online linkage is not required for EPA minimum requirements. It is highly encouraged in order to promote reuse of and access to resources. Note - ArcCatalog often populates this field with a local path to data, or the details of an ArcSDE connection. This conflicts with the FGDC Guidance as well as Data.gov requirements.  If local paths to data or ArcSDE connection info are truly appropriate to include in a metadata record (and in almost all cases, they are not appropriate), please place that information in another section, such as "Supplemental Info". 

This element should reflect the 'Data Resource Type' classification provided in Tab 3. For example, if the resource type is listed as 'Live Data and Services', the URL provided in this field should reflect a live data and service URL. An example of how to document live data and services is provided below. For more information on resource types, see the 'Data Resource Type' field in Tab 3.
At 3.2.1, to support Data.gov requirements, we are promoting the following convention, and these linkages are mandatory if applicable. If a type of link is not applicable, please leave it blank, and please include only the URL and no other text.
  • Primary Linkage: URL providing direct access to the downloadable distribution of a dataset.
  • Secondary Linkage: Endpoint of web service to access dataset. This is typically the REST endpoint for an ArcGIS Server service, but it may also be a WMS GetCapabilities URL or a SOAP WSDL endpoint.
  • Linkage 3: URL for accessing related documents such as technical information about a dataset, developer documentation, etc.
  • Linkage 4: Alternative landing page used to redirect user to a contextual, Agency-hosted “homepage” for the Dataset or API when selecting this resource from a metadata catalog. This field is not intended for an agency’s homepage (e.g. www.agency.gov), but rather if a dataset has a human-friendly hub or landing page that users should be directed to for all resources tied to the dataset. This allows agencies to better specify what a visitor receives after selecting one of the agency’s datasets on Data.gov or in third-party mashups.
  • Other Linkages: Any other URLs related to this resource

Domain

Examples:
Primary Linkage Download URL:
"https://edg.epa.gov/data/Public/OEI/EPA_Facilities/EPA_Locations.zip"
Secondary Linkage Map Service:
"http://geodata.epa.gov/arcgis/rest/services/OEI/EPA_Locations/MapServer"
