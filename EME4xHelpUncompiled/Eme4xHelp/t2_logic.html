Interface Element Description: Tab 2

Element: Logical Consistency Report (Data Integrity Testing)
Type: text
Short Name: logic

FGDC Definition

An explanation of the fidelity of relationships in the data set and tests used.

EPA Requirement/Guidance

This element is required as per minimum FGDC requirements. It is used to provide an overview of the types of tests that were performed to determine data integrity (consistency of features in the data set, as opposed to attributes or overall positional accuracy).

Types of tests are listed below (extracted from SDTS). For more information, please visit http://www.fgdc.gov/csdgmgraphical/dataq/logconfq.htm
  • Tests of Valid Values
    1. Tests for permissible values may be applied to any data structure.
    2. Such a test can detect gross blunders, but it does not ensure all aspects of logical consistency.

  • General Tests for Graphic Data
    1. A database containing lines may be subjected to the following general questions:
      1. Do lines intersect only where intended?
      2. Are any lines entered twice?
      3. Are all areas completely described?
      4. Are there any overshoots or undershoots?
      5. Are any polygons too small, or any lines too close?
    2. Different tests may be applied to address these questions.
    3. The report should contain a description of tests applied or a reference to software used.
    4. The report should state whether inconsistencies were corrected or detail the remaining errors.

  • Specific Topological Tests
    1. Logical consistency may be reported as "Topologically Clean" if an automated procedure concludes:
      1. All chains intersect at nodes (report tolerance used).
      2. Chains and nodes are consistent around polygons or vice-versa.
      3. Inner rings embed consistently in enclosing polygons.
    2. The quality report shall identify the software (name and version) used to verify these conditions.

Date of Test
The report should include testing date(s). If corrections/modifications occur after the tests, the report should indicate how the new information was checked for logical consistency.
