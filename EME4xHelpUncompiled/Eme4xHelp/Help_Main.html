EPA Metadata Editor (EME)

Introducing the EPA Metadata Editor

What is the EPA Metadata Editor?

The U.S. Environmental Protection Agency (EPA) Metadata Editor (EME) is both a desktop application and extension to the Environmental Systems Research Institute's (ESRI) ArcCatalog that is intended to simplify and expedite the process of developing geospatial metadata across the agency. The goal of the EPA Metadata Editor is to create greater geospatial metadata consistency and compliancy agency-wide in order to provide a solid framework for sharing assets with others. For more information and additional resources regarding the EME, please see https://edg.epa.gov/EME/.

What's new?

EME version 4.0 was developed to support Data.gov's Project Open Data Metadata initiative in conjunction with the transition to ISO 19115 Parts 1 and 2. It is currently recommended that EME version 4.0 be used for non-spatial datasets and EME version 3.2.1 be used for spatial datasets.

EME version 4.0 is more streamlined than version 3.2.1. Some of the highlights of version 4.0 include: less data entry, ability to add multiple contacts per metadata record, and automated validation.

Background

  • EME version 2.1 was initially released in July 2007. This release was deployed with a number of key features including a configurable database, easily accessible help, an EPA validation service, and a number of interface features designed to help users easily enter information into their records.

  • EME version 3 was released in October 2008, and provided a number of new capabilities, including a configurable EPA Synchronizer that allows users to control synchronization, a configurable path to the EME database, batch validation, ability to remove ESRI tags, and more.

  • EME version 3.1 was released in April 2011 in order to provide users with an ArcGIS 10-compatible version of the application. This update was required as a result of changes in the ArcGIS 10 framework that made it incompatible with EME 3.0 and EME 2.1. Additionally, much of the functionality for supporting FGDC CSDGM metadata was removed in ArcGIS version 10. As such, much of the underlying code base for the EME was updated to be compatible with ArcGIS version 10, and at the same time the application was released as a standalone product. EME 3.1.1 now provides users with the ability to edit and synchronize FGDC CSDGM metadata within the ArcGIS 10 framework and to run the product as a standalone tool.

  • EME version 3.1.1 was released in July 2011. It provided users with a metadata export tool, keyboard shortcuts for copy and paste, and several other minor enhancements and bug fixes.

  • EME version 3.1.2 was released in September 2011. It provided users with a global find and replace function (single or batch processing) as well as key bug fixes for installing EME and "Refresh from Database" functionality.

  • EME version 3.2 was released in November 2012. It adds support for several fields (Spatial Data Organization Information, Process Step Contact Information, and Source Information) and two projections (Albers Conical Equal Area and Web Mercator Auxiliary Sphere). This release also includes minor updates to the EPA stylesheet and the syncrhonization process. It fixes bugs that affected validation and XML export.

  • EME version 3.2.1 was released in October 2014. It provides users with additional Online Linkage elements, dataset Access Constraint options, Constraint Security Classification options, and implements mandatory elements for Distribution Contact including Publisher, Contact Name, and Contact Email.
