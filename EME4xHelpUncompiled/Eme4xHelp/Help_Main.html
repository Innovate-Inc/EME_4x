EPA Metadata Editor (EME)

Introducing the EPA Metadata Editor

What is the EPA Metadata Editor?

The U.S. Environmental Protection Agency (EPA) Metadata Editor (EME) is a desktop application that is intended to simplify and expedite the process of developing geospatial metadata across the agency. The program is built as an extension to the Environmental Systems Research Institute's (ESRI) ArcCatalog and, as of EME version 3.1, also runs as a standalone application. The EME is provided as a customized metadata editing tool that allows users to create metadata that adheres to the EPA's implementation of the Federal Geographic Data Committee's (FGDC's) Content Standard for Digital Geospatial Metadata (CSDGM).

It is hoped that the use of the EPA Metadata Editor will result in greater geospatial metadata consistency and compliancy agency-wide in order to provide a solid framework for sharing assets with others. For more information and additional resources regarding the EME, please see https://edg.epa.gov/EME/.

What's new?

The latest release of EME is version 3.2. This version includes the following updates:

  • EME now supports Spatial Data Organization Information, Source Information, and Process Step Contact Information.
  • EME now supports additional projections: Albers Conical Equal Area and Web Mercator Auxiliary Sphere.
  • The EME download package now includes an EPA-specific stylesheet
  • A warning now appears before synchronization.
  • Bug fix: Addressed a bug associated with validation.
  • Bug fix: Addressed a bug that stripped XML header information during export.

    • EME version 3.1.2. was released in November, 2011 and included the following updates:

  • Provided users with a Global Find and Replace function from the EME Toolbar
  • Bug fix: Addressed installation issues identified in EME 3.1.1
  • Bug fix: Addressed a bug associated with using the "Refresh From Database" Function within Tab 2

    • EME version 3.1.1 was released July 27, 2011 and included the following updates:

  • Provides users with a Metadata Export tool
  • Provides users with keyboard shortcuts for copy/cut/paste in the EME user interface
  • Save and close warning has been removed from pop-up windows
  • Provides access to EME help files directly from EME Toolbar
  • Bug fix: Detailed attribute information is no longer erased when using the EPA synchronizer with Feature Class Export and/or Feature Class Copy/Paste
  • Bug fix: When user selects multiple processing steps for deletion, EME no longer only deletes the first step upon save.
  • Bug fix: User can now choose to open a metadata record and immediately select "Save As" without requiring that the user modify the open record first.

    • EME version 3.1 was the last major release (4/1/2011) and included the following updates:

  • Provided users with an FGDC CSDGM metadata editor that works in ArcGIS version 10
  • Provided users with the ability to install a standalone version of the application that works outside the ArcGIS environment
  • Provided users with the ability to synchronize FGDC metadata within ArcGIS version 10
  • Provided users with the ability to include more than two online linkages in their metadata
  • Provided users with a 'Clear All Metadata' button to remove all content from their record
  • Provided users with an Import tool, accessible from ArcCatalog, that allows users to select the object from which they want to import metadata
  • The installation location of the EME database has been moved to a location that is not typically write-protected (for individuals that may have managed desktop environments)

    EME version 3.1 also provided the following bug fixes:

  • FGDC CSDGM section 6 is no longer accidentally erased from user's metadata record in special cases
  • XML header information is no longer removed from user's metadata record
  • Processing contribution information is no longer removed from user's metadata record
  • Installing and using EME under different user accounts no longer generates an error when EME looks for the last_session.xml file.

    • Background

  • EME version 2.1 was initially released in July 2007. This release was deployed with a number of key features including a configurable database, easily accessible help, an EPA validation service, and a number of interface features designed to help users easily enter information into their records.

  • EME version 3 was released in October 2008, and provided a number of new capabilities, including a configurable EPA Synchronizer that allows users to control synchronization, a configurable path to the EME database, batch validation, ability to remove ESRI tags, and more.

  • EME version 3.1 was released in April 2011 in order to provide users with an ArcGIS 10-compatible version of the application. This update was required as a result of changes in the ArcGIS 10 framework that made it incompatible with EME 3.0 and EME 2.1. Additionally, much of the functionality for supporting FGDC CSDGM metadata was removed in ArcGIS version 10. As such, much of the underlying code base for the EME was updated to be compatible with ArcGIS version 10, and at the same time the application was released as a standalone product. EME 3.1.1 now provides users with the ability to edit and synchronize FGDC CSDGM metadata within the ArcGIS 10 framework and to run the product as a standalone tool.

  • EME version 3.1.1 was released in July 2011. It provided users with a metadata export tool, keyboard shortcuts for copy and paste, and several other minor enhancements and bug fixes.

  • EME version 3.1.2 was released in September 2011. It provided users with a global find and replace function (single or batch processing) as well as key bug fixes for installing EME and "Refresh from Database" functionality.

  • EME version 3.2 was released in November 2012. It adds support for several fields (Spatial Data Organization Information, Process Step Contact Information, and Source Information) and two projections (Albers Conical Equal Area and Web Mercator Auxiliary Sphere). This release also includes minor updates to the EPA stylesheet and the syncrhonization process. It fixes bugs that affected validation and XML export.
