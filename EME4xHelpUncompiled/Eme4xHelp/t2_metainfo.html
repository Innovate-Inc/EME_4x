EPA Metadata Editor (EME)

Interface Element Description: Metadata Information

Mandatory element

EPA Requirement/Guidance

This section contains additional information about the metadata.

Metadata Information Section:

  • Type of Resource - The class of information to which the referencing entity applies.
  • Last Modified - Date of the metadata creation or the last metadata update.
  • Identifier - Unique identifier for metadata record.
  • Language - Language of the metadata composed of an ISO639-2/T three letter language code and an ISO3166-1 three letter country code.
  • Standard Name - The name of the metadata standard/profile use.
  • Standard Version - The version of the metadata standard/profile used.

Type of Resource: The class of information to which the referencing entity applies. Best practice: EME version 4.0 will primarily be used for "non-geographic datasets"; however, "datasets" may also be selected for any data resources with a spatial component.

Last Modified: Date of the metadata creation or the last metadata update.

Identifier: This element is the universally unique identifier (UUID) for the metadata record. The UUID provides the link between the metadata associated with a resource (e.g., feature class) in your local system to the metadata uploaded and published with the EDG. If a computer (or person) knows the UUID of a metadata record published with the EDG, then a generic URL + UUID can be used to view that record (i.e., https://edg.epa.gov/metadata/catalog/search/resource/details.page?uuid={0CA7A9AE-68DB-4F92-8566-2003C8BF41AB}). Many applications leverage generic URL + UUID. For example, when services are published, users will place the EDG UUID URL inside the MXD for each data layer. This way you can get access to the metadata via a REST endpoint (or web application) for a given data resource.

Best practice: If the user has a new metadata record (never before published with EDG) then the user will want to create a New Identifier by clicking on the “New Identifier” button. By creating a New Identifier, the user will avoid the hassle of having to return and embed in the record the UUID assigned by the EDG after upload. (The user could possibly also get a head start on inserting links into their published data, MXD's for published services, etc). If the user leaves the Identifier element blank, the EDG will assign a UUID upon successful upload.

Users should be aware that if they override an existing UUID, the link will be severed between their local metadata and the metadata record stored at the EDG.  If this happens, then during the publishing process the EDG will create a new metadata record rather than changing the existing record.

Language: Although ISO has a specific format for indicating language (described below), Data.gov references a different specification. Until these two are reconciled, EPA records should adhere to the RFC 5646 standard. This language subtag lookup provides a good tool for checking and verifying language codes. A language tag is comprised of either one or two parts, the language subtag (such as en for English, sp for Spanish, wo for Wolof) and the regional subtag (such as US for United States, GB for Great Britain, MX for Mexico), separated by a hyphen. Regional subtags should only be provided when needed to distinguish a language tag from another one (such as American vs. British English).

Example: en-US or if multiple languages, "es-MX","wo","nv","en-US"

ISO specifies that language code and country code be documented in the following manner: < ISO639-2/T three letter language code > < ; > < blank space > < ISO3166-1 three letter country code > Country code is given in uppercase.

Example: eng; USA

Standard Name: The name of the metadata standard/profile use. Typically, the value for this field will be:

  • ISO 19115: "ISO 19115 Geographic information - Metadata"
  • ISO 19115-2: "ISO 19115-2 Geographic Information - Metadata - Part 2: Extensions for Imagery and Gridded Data"s

Standard version: The version of the metadata standard/profile used. Typically, the value for this field will be:

  • ISO 19115: "ISO 19115:2003(E)"
  • ISO 19115-2: "ISO 19115-2:2009(E)"

Domain

Type of Resource:

  • dataset - information applies to the dataset
  • nonGeographicDataset - information applies to non-geographic data
  • attribute - information applies to the attribute class
  • attributeType - information applies to the characteristic of a feature
  • collectionHardware - information applies to the collection hardware class
  • collectionSession - information applies to the collection session
  • series - information applies to the series
  • service - information applies to a capability which a service provider entity makes available to a service user entity through a set of interfaces that define a behaviour, such as a use case
  • dimensionGroup - information applies to a dimension group
  • feature - information applies to a feature
  • featureType - information applies to a feature type
  • propertyType - information applies to a property type
  • fieldSession - information applies to a field session
  • software - information applies to a computer program or routine
  • model - information applies to a copy or imitation of an existing or hypothetical object
  • tile - information applies to a tile, a spatial subset of geographic data
